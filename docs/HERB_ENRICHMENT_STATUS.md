## Metadata Template for New Herbs

Use this template when enriching new herbs:

```json
{
  "properties": {
    "magical": ["property1", "property2", "property3"],
    "medicinal": ["benefit1", "benefit2", "benefit3"],
    "spiritual": ["aspect1", "aspect2", "aspect3"]
  },
  "preparations": {
    "primary": [
      "Method 1: description",
      "Method 2: description"
    ],
    "alternative": ["Alt method"],
    "dosage": "Specific amounts"
  },
  "associations": {
    "deities": ["deity_id"],
    "concepts": ["concept"],
    "elements": ["element"],
    "chakras": ["chakra"]
  },
  "harvesting": {
    "season": "When",
    "method": "How",
    "conditions": "Where/Requirements"
  },
  "dangers": {
    "toxicity": "Level",
    "warnings": ["Warning"],
    "contraindications": ["Contraindication"]
  },
  "substitutes": [
    {
      "name": "Alternative herb",
      "reason": "Why works as substitute",
      "tradition": "tradition_context"
    }
  ],
  "botanicalInfo": {
    "scientificName": "Genus species",
    "family": "Family name",
    "nativeRegion": "Geographic origin",
    "commonNames": ["Name1", "Name2"]
  }
}
```

## Contributing Guidelines

When enriching a new herb:

1. **Research thoroughly**
   - Use primary and secondary sources
   - Verify botanical information
   - Document all sources

2. **Follow the schema**
   - Use exact format from HERB_METADATA_SCHEMA.md
   - Fill all 7 sections
   - Maintain consistency

3. **Validate quality**
   - Run: `node scripts/validate-herb-metadata.js herb_id`
   - Aim for 85%+ completeness
   - Fix all errors before submission

4. **Test integration**
   - Run enrichment script
   - Verify in Firebase Console
   - Test in application views

5. **Document changes**
   - Note sources used
   - Update this status report
   - Add examples if new patterns
