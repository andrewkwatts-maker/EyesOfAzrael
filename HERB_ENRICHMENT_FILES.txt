================================================================================
THE 7 METADATA SECTIONS (SUMMARY)
================================================================================

1. PROPERTIES
   Fields: magical[], medicinal[], spiritual[]
   Purpose: Categorize herb characteristics
   Template: 3-7 items per category

2. PREPARATIONS
   Fields: primary[], alternative[], dosage
   Purpose: Provide usage instructions
   Template: 2-5 methods + alternatives + dosage

3. ASSOCIATIONS
   Fields: deities[], concepts[], elements[], chakras[]
   Purpose: Link to other entities/concepts
   Template: Entity IDs, terms, elements

4. HARVESTING
   Fields: season, method, conditions
   Purpose: Ethical collection guidance
   Template: Timing, practice, requirements

5. DANGERS
   Fields: toxicity, warnings[], contraindications[]
   Purpose: Ensure safe use
   Template: Toxicity level + precautions

6. SUBSTITUTES
   Fields: [{ name, reason, tradition }]
   Purpose: Alternative options
   Template: 1-3 alternatives

7. BOTANICAL INFO
   Fields: scientificName, family, nativeRegion, commonNames[]
   Purpose: Scientific classification
   Template: Taxonomic data

   Command: node scripts/validate-herb-metadata.js

2. ENRICH
   Command: export FIREBASE_SERVICE_ACCOUNT=./serviceAccountKey.json
            node scripts/enrich-herbs-metadata.js

3. VERIFY
   Action: Open Firebase Console → Firestore → herbs

4. EXPAND
   File: scripts/enrich-herbs-metadata.js
   Edit: Add more herbs to HERB_ENRICHMENT_DATA
